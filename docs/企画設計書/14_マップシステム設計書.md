## 3. ダンジョン生成（BSP法）

### 3.1 参照ファイル

- `src/RougelikeGame.Core/Map/Generation/DungeonGenerator.cs`
- `src/RougelikeGame.Core/Map/Generation/BSPNode.cs`（同ファイル内）
- `src/RougelikeGame.Core/Map/Generation/RoomGenerator.cs`
- `src/RougelikeGame.Core/Map/Generation/CorridorGenerator.cs`

### 3.2 生成フロー

```
Generate(parameters)
  │
  ├─ 1. GenerateRooms（BSP空間分割 → 部屋配置）
  │   ├─ BSPTree作成（マージン2タイル）
  │   ├─ 再帰分割（最大深度6、最小サイズ8）
  │   ├─ リーフノードに部屋を作成
  │   │   ├─ 70%: 矩形部屋
  │   │   ├─ 20%: 円形部屋
  │   │   └─ 10%: 十字形部屋
  │   └─ 入口部屋・ボス部屋を決定
  │       └─ 入口から最も遠い（チェビシェフ距離）部屋をボス部屋に
  │
  ├─ 2. ConnectRooms（最小スパニングツリー + 追加接続）
  │   ├─ MSTで全部屋を接続
  │   ├─ 追加接続（rooms/3 本のループ）
  │   └─ 廊下種別:
  │       ├─ L字型（ConnectRoomsL）
  │       ├─ ジグザグ型（ConnectRoomsZigzag）
  │       └─ 直線型（ConnectRoomsDirect）
  │
  ├─ 3. FixDiagonalOnlyTiles（1回目）
  │   └─ 斜めのみで到達可能なタイルを修正
  │
  ├─ 4. PlaceDoors（25%の確率）
  │   └─ 部屋と廊下の接続点にドアを配置
  │
  ├─ 5. PlaceStairs
  │   ├─ 入口部屋に上り階段
  │   └─ ボス部屋に下り階段
  │
  ├─ 6. DecorateRooms
  │   ├─ 宝物庫（Treasure）を割り当て
  │   ├─ 祭壇の間（Shrine）を割り当て
  │   └─ 各部屋に装飾タイル配置
  │
  ├─ 7. PlaceTraps（TrapDensity比率）
  │   └─ 床タイルに隠し罠を配置
  │
  └─ 8. FixDiagonalOnlyTiles（2回目）
      └─ 装飾・罠配置後の再修正
```

### 3.3 BSP分割パラメータ

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 最小ノードサイズ | 8 | これ以下に分割しない |
| 最大分割深度 | 6 | 再帰の上限 |
| 分割確率 | 1.0 - depth × 0.15 | 深くなるほど分割しにくい |
| 部屋最小サイズ | 5×5 | ノード内マージン2タイル |
| 部屋最大サイズ | 12×10 | ノードサイズによる制約 |
| ノードマージン | 2 | ノード端からの部屋配置マージン |

### 3.4 FixDiagonalOnlyTiles

斜めのみでアクセス可能なタイルを4方向移動で到達可能にする後処理:

1. **孤立タイル除去**: 歩行可能タイルの上下左右が全て壁 → 壁に変換
2. **斜め接続修正**: 斜めに床があるが間の2タイルが両方壁 → 一方を廊下に変換
3. **反復**: 変更がなくなるまで（最大10回）

## 4. 部屋システム

### 4.1 部屋種別（RoomType）

| RoomType | 説明 | 決定方法 |
|----------|------|---------|
| Normal | 通常の部屋 | デフォルト |
| Entrance | 入口部屋（上り階段あり） | 最初に生成された部屋 |
| Boss | ボス部屋（下り階段あり） | 入口から最も遠い部屋 |
| Treasure | 宝物庫 | 通常部屋からランダム選択 |
| Shrine | 祭壇の間 | 通常部屋からランダム選択 |

### 4.2 部屋形状

| 形状 | 確率 | 説明 |
|------|------|------|
| 矩形 | 70% | 標準的な長方形の部屋 |
| 円形 | 20% | 楕円形に近い部屋 |
| 十字形 | 10% | 十字型に掘られた部屋 |

## 5. 廊下システム

### 5.1 接続アルゴリズム

1. **最小スパニングツリー (MST)**: チェビシェフ距離を基準に全部屋を最短経路で接続
2. **追加接続**: ランダムに `rooms/3` 本のループ経路を追加（行き止まり防止）

### 5.2 廊下形状

| 形状 | 選択確率 | 説明 |
|------|---------|------|
| L字型 | 33% | 水平→垂直（または逆）で接続 |
| ジグザグ型 | 33% | 段階的に方向転換しながら接続 |
| 直線型 | 33% | ブレゼンハム的に直線接続 |

### 5.3 ドア配置

- 部屋と廊下の接続点に **25%** の確率でドアを配置
- 閉じたドアは接触で自動的に開く

## 7. FOV（視野計算）

| 項目 | 値 |
|------|-----|
| 視野半径 | 8 タイル（`DefaultViewRadius`） |
| アルゴリズム | DungeonMap.ComputeFov |
| 遮断条件 | `BlocksSight = true` のタイル |
| 探索記録 | 一度でも視界に入ったタイルは `IsExplored = true` に永続化 |
| 描画への影響 | 可視→通常色、探索済み→暗い色、未探索→黒 |

## 8. 自動探索（BFS）

| 項目 | 値 |
|------|-----|
| アルゴリズム | 幅優先探索（BFS） |
| 目標 | 最寄りの未探索タイルへの隣接歩行可能タイル |
| 実行間隔 | 80ms（DispatcherTimer） |
| 停止条件 | 敵発見、アイテム発見、階段発見、HP低下、手動入力 |
| 操作キー | Tab（開始/停止トグル） |
