## 3. ダンジョン生成（BSP法）

### 3.1 参照ファイル

- `src/RougelikeGame.Core/Map/Generation/DungeonGenerator.cs`（BSPNode も同ファイル内に定義）
- `src/RougelikeGame.Core/Map/Generation/RoomCorridorGenerator.cs`（部屋・廊下生成）
- `src/RougelikeGame.Core/Map/Generation/FeaturePlacer.cs`（装飾・罠・ドア配置）

### 3.2 生成フロー

```
Generate(parameters)
  │
  ├─ 0. リトライループ（MaxRetries=5、MinRooms=5保証）
  │   └─ 部屋数 < MinRooms または階段未配置なら再生成（forceMinRooms=true）
  │
  ├─ 1. GenerateRooms（BSP空間分割 → 部屋配置）
  │   ├─ BSPTree作成（マージン2タイル）
  │   ├─ 再帰分割（最大深度6、最小サイズ8）
  │   ├─ リーフノードに部屋を作成
  │   │   ├─ 70%: 矩形部屋
  │   │   ├─ 20%: 円形部屋
  │   │   └─ 10%: 十字形部屋
  │   └─ 入口部屋・ボス部屋を決定
  │       └─ 入口から最も遠い（チェビシェフ距離）部屋をボス部屋に
  │
  ├─ 2. ConnectRooms（最小スパニングツリー + 追加接続）
  │   ├─ MSTで全部屋を接続
  │   ├─ 追加接続（rooms/3 本のループ）
  │   └─ 廊下種別:
  │       ├─ L字型（ConnectRoomsL）
  │       ├─ ジグザグ型（ConnectRoomsZigzag）
  │       └─ 直線型（ConnectRoomsDirect）
  │
  ├─ 3. FixDiagonalOnlyTiles（1回目）
  │   └─ 斜めのみで到達可能なタイルを修正
  │
  ├─ 4. PlaceDoors（25%の確率）
  │   └─ 部屋と廊下の接続点にドアを配置
  │
  ├─ 5. PlaceStairs
  │   ├─ 入口部屋に上り階段
  │   └─ ボス部屋に下り階段
  │
  ├─ 6. DecorateRooms
  │   ├─ 宝物庫（Treasure）を割り当て
  │   ├─ 祭壇の間（Shrine）を割り当て
  │   ├─ 各部屋に装飾タイル配置
  │   └─ WouldBlockPassage() で通路を塞ぐ位置には柱・噴水を配置しない
  │
  ├─ 7. PlaceTraps（TrapDensity比率）
  │   └─ 床タイルに隠し罠を配置
  │
  └─ 8. FixDiagonalOnlyTiles（2回目）
      └─ 装飾・罠配置後の再修正
```

### 3.3 BSP分割パラメータ

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 最小ノードサイズ | 9 | これ以下に分割しない（Ver.prt.0.18で8→9に変更、Margin×2+MinRoomSize=4+5=9を保証） |
| 最大分割深度 | 6 | 再帰の上限 |
| 分割確率 | 1.0 - depth × 0.15 | 深くなるほど分割しにくい |
| forceMinRooms深度閾値 | depth < 4 | リトライ時の強制分割閾値（Ver.prt.0.18で<3→<4に強化） |
| 部屋最小サイズ | 5×5 | ノード内マージン2タイル（壁含む、実質3×3歩行可能領域）（Ver.prt.0.18で7→5に変更） |
| 部屋最大サイズ | 12×10 | ノードサイズによる制約 |
| ノードマージン | 2 | ノード端からの部屋配置マージン |
| 最低部屋数 | 5 | MinRooms。これ未満の場合リトライまたはフォールバック配置 |
| 最大リトライ回数 | 5 | BSP生成を最大5回リトライ |
| フォールバック部屋配置 | 最大100回試行 | リトライ最終回でもMinRooms未満の場合にランダム部屋を追加配置（Ver.prt.0.18で新規追加） |

### 3.4 FixDiagonalOnlyTiles

斜めのみでアクセス可能なタイルを4方向移動で到達可能にする後処理:

1. **孤立タイル除去**: 歩行可能タイルの上下左右が全て壁 → 壁に変換
2. **斜め接続修正**: 斜めに床があるが間の2タイルが両方壁 → 一方を廊下に変換
3. **反復**: 変更がなくなるまで（最大10回）

## 5. 廊下システム

### 5.1 接続アルゴリズム

1. **最小スパニングツリー (MST)**: チェビシェフ距離を基準に全部屋を最短経路で接続
2. **追加接続**: ランダムに `rooms/3` 本のループ経路を追加（行き止まり防止）

### 5.2 廊下形状

| 形状 | 選択確率 | 説明 |
|------|---------|------|
| L字型 | 33% | 水平→垂直（または逆）で接続 |
| ジグザグ型 | 33% | 段階的に方向転換しながら接続 |
| 直線型 | 33% | ブレゼンハム的に直線接続 |

### 5.3 ドア配置

- 部屋と廊下の接続点に **25%** の確率でドアを配置
- 閉じたドアは接触で自動的に開く
- **HasWalkableBeyondWallバリデーション**: ドアの通路方向の先に実際に通行可能タイルが存在するか検証し、壁際の無意味なドア配置を防止（Ver.prt.0.10）

### 11.4 生成フロー

```
Generate(territory)
  │
  ├─ 1. GetTerritoryMapSize() で領地別サイズ取得
  │
  ├─ 2. GenerateShapeMask() で複雑形状マスク生成
  │   ├─ ノイズベース楕円（中心からの距離+角度ノイズ）
  │   ├─ 突起/湾で不規則な形状
  │   └─ 隣接領地と同一シードで境界一致
  │
  ├─ 3. FillBaseTerrain（形状マスク内を領地地形で埋める）
  │   ├─ 60%: 主要地形（Grass/Forest/Mountain等）
  │   ├─ 25%: 副次地形
  │   └─ 15%: 障害物
  │
  ├─ 4. PlaceSettlements（村/町/都の自動配置）
  │   ├─ 都: 1箇所/領地（マップ中心付近に優先配置）
  │   ├─ 町: 総マス数÷1000箇所
  │   ├─ 村: 総マス数÷500箇所
  │   └─ 相互距離12-25マス以上（Chebyshev距離）
  │
  ├─ 5. PlaceRandomDungeons（ランダムダンジョン配置）
  │   ├─ 集落から25マス以上離れた位置（元の50マスの1/2）
  │   ├─ 他ダンジョンから50マス以上離れた位置（元の100マスの1/2）
  │   ├─ 野盗のねぐら / ゴブリンの巣（1-3階層）
  │   └─ クリア後消滅
  │
  ├─ 6. ConnectLocationsWithRoads（道路接続）
  │   └─ ロケーション間を歩行可能な道で接続
  │
  └─ 7. SetEntrance（入口位置設定）
```

### 11.7 ロケーション入場操作

全ロケーション（町・村・都・施設・神殿・フィールド・ダンジョン）への入場は **Tキー**（`TryEnterTown`）に統一。
シンボルマップ上のロケーションシンボルに隣接し、Tキーを押すことで入場する。

> **Ver.prt.0.15 変更**: ダンジョンのみ `>` キー（TryDescendStairs）で入場する仕様だったものをTキーに統一。TryDescendStairs は地上シンボルマップでは「Tキーで入場してください」のガイダンスメッセージを表示するよう変更。

### 11.8 シンボルマップ上アクション制限（Ver.prt）

シンボルマップ上では以下のアクションのみ許可される：

| 許可されるアクション | 説明 |
|-------------------|------|
| 移動（8方向） | MoveUp/Down/Left/Right/斜め |
| 待機 | Wait |
| インベントリ | OpenInventory（アイテム使用・装備変更含む） |
| ステータス・ログ確認 | OpenStatus, OpenMessageLog |
| ロケーション進入 | EnterTown, UseStairs, AscendStairs |
| ワールドマップ（情報参照のみ） | OpenWorldMap（A2: 移動機能は廃止、関所経由に統一） |
| 町離脱 | LeaveTown |

> 攻撃・スキル使用・自動探索・祈りなどのアクションはシンボルマップ上では使用不可。
> 領地間移動はワールドマップからの直接移動ではなく、関所（BorderGate）のNPCにインタラクトして行う（A2仕様）。

### 11.9 勢力影響と敵種別（Ver.α.0.2 / A1仕様追加）

`TerritoryInfluenceSystem` により、シンボルマップ上の各タイルに影響力の高い勢力が設定され、
フィールドに進入した際の出現敵がこの勢力に応じて変化する。

#### 安全圏/危険圏の重複処理ルール（A1仕様）

- **安全圏同士の重複**: そのまま処理（安全圏のまま）
- **安全圏と危険圏の重なり**: 日数経過ごとに危険圏が拡大する（30日ごとに安全圏距離が1マス縮小、最小は元の半分）
- **危険圏同士の重複**: 派閥勢力（Influence値）によって支配範囲の縮小/拡大が決定
- **BorderGate**: 常に安全圏として扱う
- **Dungeon/BanditDen/GoblinNest**: 圏域判定対象外（安全圏を生成しない）

#### 安全圏距離

- 集落（村/町/都）および施設（Facility/ReligiousSite）から距離内: 野生動物（安全圏）
- 安全圏距離: マップ対角線の1/10（最低10マス、最大50マス、デフォルト30マス）
- 日数経過で安全圏距離が縮小（`DangerExpansionDaysPerTile = 30`日ごとに1マス）

#### 領地間移動（A2仕様）

- ワールドマップからの直接移動は廃止
- 関所（BorderGate）のNPCにインタラクト（Tキー）して移動が成立
- 通行料（100G）、手配チェック、戦争状態チェック、評判チェックは関所で実施
- 旅路イベントは関所経由の移動時にも発生

## 12. ロケーション別マップ生成

### 12.1 参照ファイル

- `src/RougelikeGame.Core/Map/Generation/LocationMapGenerator.cs`

### 12.2 概要

シンボルマップ上のロケーションに進入した際に生成される個別マップ。
ロケーションタイプ（LocationType）に応じた形状のマップを生成する。

### 12.3 ロケーションタイプ別マップ

| タイプ | サイズ | 特徴 |
|--------|-------|------|
| Town | 50×30 | 十字道路、中央噴水広場、四隅と道路沿いに建物。NPC配置：訓練師(NpcTrainer)・図書館司書(NpcLibrarian)・商人(NpcShopkeeper) |
| Village | 40×25 | 横道、小屋3棟、畑エリア、井戸 |
| Facility | 30×20 | 大部屋メインホール、仕切り壁、カウンター、祭壇 |
| ReligiousSite | 35×25 | 回廊付き神殿、中央祭壇、瞑想小部屋 |
| Field | 60×40 | 自然地形（草地・木・水）にランダムな空き地 |

### 12.4 外周・脱出仕様

- **Town / Village**: 外周はGrass（通行可能）。壁ボーダーなし
- **Facility / ReligiousSite**: FillAll(Wall)後にDrawPassableBorder()で外周をFloor（通行可能）に設定
- **Field**: 外周は自然生成のまま（Grass基調、一部Tree/Water）
- **脱出方法**: マップ端タイルに立ち、外方向に移動キーを押すとTryMove()の境界チェック→TryLeaveTown()が発動し、シンボルマップに帰還
- **StairsUp**: 全ロケーションマップに配置される上り階段（スポーンポイント兼脱出手段）
  - Town/Village/Field: SetEntrance() + SetStairsUp() で入口座標に配置
  - Facility/ReligiousSite: PlaceExitDoor() で出口ドアの先に配置
