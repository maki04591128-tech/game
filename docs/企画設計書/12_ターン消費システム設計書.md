## 1. システム概要

### 1.1 コンセプト

本作のターンシステムは「1ターン = 1秒」を基本単位とし、行動の種類や状況によって消費ターンが変動する。
これにより、リアルな時間経過とゲームプレイのテンポを両立させる。

```
┌─────────────────────────────────────────────────────────────┐
│                    ターン消費システム概念図                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1ターン = 1秒（ゲーム内時間）                               │
│                                                             │
│   【状況別ターン消費】                                        │
│                                                             │
│   通常移動（1マス）        : 1ターン  = 1秒                   │
│   戦闘中/隠密中移動（1マス）: 10ターン = 10秒                  │
│   攻撃（通常攻撃）         : 1ターン  = 1秒                   │
│   スキル使用              : 各スキル定義 = 可変               │
│   魔法詠唱                : 語数依存 = 5~100秒               │
│                                                             │
│   これにより:                                                │
│   ・戦闘は緊張感のある「遅い」時間になる                      │
│   ・探索時は軽快に移動できる                                 │
│   ・時間制限イベントでの戦略性が増す                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 設計思想

| 原則 | 説明 |
|------|------|
| リアリティ | 戦闘中の慎重な動きは時間がかかる |
| 戦略性 | 行動選択にターン消費を考慮させる |
| テンポ | 非戦闘時は快適に移動できる |
| 時間管理 | 限られた時間でのリソース管理 |

## 3. 状況遷移

### 3.1 戦闘状態への移行

```
[通常状態]
    │
    ├─ 敵を視認 → [警戒状態] (Alert)
    │               │
    │               ├─ 敵が接近 → [戦闘状態] (Combat)
    │               │
    │               └─ 敵を見失う(30ターン) → [通常状態]
    │
    └─ 敵から攻撃を受ける → [戦闘状態] (Combat)

[戦闘状態]
    │
    ├─ 敵を全滅させる → [警戒状態] (30ターン) → [通常状態]
    │
    └─ 一定距離逃走 → [追跡状態] (Pursuit) → [警戒状態] → [通常状態]
```

### 3.2 隠密状態

```
[通常状態]
    │
    └─ 隠密スキル発動 → [隠密状態] (Stealth)
                          │
                          ├─ 敵の視界に入る → 発見判定
                          │   ├─ 成功 → [隠密維持]
                          │   └─ 失敗 → [戦闘状態]
                          │
                          ├─ 攻撃する → [戦闘状態] (奇襲ボーナス)
                          │
                          └─ SP切れ/解除 → [通常状態]
```

### 5.2 行動順決定

```
行動順 = 前回行動ターン + 今回行動消費ターン

例: 
  プレイヤー: ターン0で攻撃(3ターン消費) → 次はターン3
  ゴブリン: ターン0で攻撃(2ターン消費) → 次はターン2

  ターン2: ゴブリンが先に行動
  ターン3: プレイヤーが行動
```

## 8. 実装仕様

> **📌 現在の実装**: `TurnAction` 構造体は `src/RougelikeGame.Core/ValueObjects/TurnAction.cs` に実装済み。
> `CalculateFinalCost()` メソッドで状況修正・装備修正・状態異常修正・環境修正・満腹度/渇き度デバフ加算を計算。
> `GameController.ProcessInput()` でターン消費定数を適用し、行動ごとのコスト処理を実行。

### 8.1 TurnAction構造体（実装済み）

```csharp
public struct TurnAction
{
    public TurnActionType Type { get; init; }
    public int BaseTurnCost { get; init; }
    public Direction Direction { get; init; }
    public IDamageable? Target { get; init; }
    public string? SkillId { get; init; }
    public string? Incantation { get; init; }
    public string? ItemId { get; init; }

    // 最終ターン消費計算（Ver.prt.0.25で満腹度/渇き度デバフ引数追加）
    public int CalculateFinalCost(
        CombatState combatState,
        float equipmentModifier = 1.0f,
        float statusModifier = 1.0f,
        float environmentModifier = 1.0f,
        int hungerCostBonus = 0,
        int thirstCostBonus = 0)
    {
        float modifier = 1.0f;

        // 移動時: 状況修正 + 斜め移動補正
        if (Type == TurnActionType.Move)
        {
            modifier *= GetMovementMultiplier(combatState);
            if (Direction.IsDiagonal())
                modifier *= (float)TurnCosts.DiagonalNumerator / TurnCosts.DiagonalDenominator;
        }

        // 接敵状態: 非戦闘行動 ×2
        if (combatState.HasFlag(CombatState.Combat) && !IsCombatAction(Type) && Type != TurnActionType.Move)
            modifier *= TurnCosts.EngagedNonCombatMultiplier;
        // 隠密状態: ドア開閉等 ×5
        else if (combatState.HasFlag(CombatState.Stealth) && IsStealthPenaltyAction(Type) && Type != TurnActionType.Move)
            modifier *= TurnCosts.StealthMovementMultiplier;

        modifier *= equipmentModifier * statusModifier * environmentModifier;

        int baseCost = Math.Max(1, (int)Math.Ceiling(BaseTurnCost * modifier));
        return Math.Max(1, baseCost + hungerCostBonus + thirstCostBonus);
    }

    // 戦闘行動判定（接敵×1）
    public static bool IsCombatAction(TurnActionType type) => type is
        TurnActionType.Attack or TurnActionType.UseSkill or TurnActionType.CastSpell or TurnActionType.UseItem;

    // 隠密ペナルティ行動判定（移動・ドア開閉）
    public static bool IsStealthPenaltyAction(TurnActionType type) => type is
        TurnActionType.Move or TurnActionType.Interact;
}
```

## 10. ゲーム全体のターン制限

### 10.1 基本制限

| 項目 | 値 |
|------|-----|
| 制限ターン数 | 31,536,000ターン（1年） |
| ゲーム内換算 | 365日（12ヶ月 × 30日 + 5日） |
| リセット条件 | 死に戻り発動時に1日目へリセット |
| 制限超過時 | 素性ごとの世界的危機が成就しゲームオーバー |

### 10.2 素性別の制限背景

ターン制限は素性ごとに異なる「世界的危機」として物語に組み込まれている。

| 素性 | 危機の種類 | 概要 |
|------|-----------|------|
| 放浪者 | ダンジョンのスタンピード | 魔物が地上へ溢れ出す |
| 復讐者 | 仇敵の逃亡 | 仇敵が国外へ逃亡する |
| 学者 | 古代遺跡の崩壊 | 手がかりが永久に失われる |
| 亡命者 | 追手の包囲網完成 | 捕縛・処刑される |
| 聖職者 | 裂け目の拡大 | 異界に世界が呑まれる |
| 商人 | 経済崩壊 | 全領地の経済が破綻する |
| 密偵 | クーデター決行 | 王国が滅ぶ |
| 罪人 | 処刑執行 | 猶予期間の終了 |
| 傭兵 | 大戦争の勃発 | 領地間の全面戦争 |
| 預言者 | 終末の預言成就 | 世界の終末 |

> **詳細**: 各素性の危機背景・延長フラグ・撤廃条件は `docs/企画設計書/18_設定資料.md` セクション7.4 を参照

### 10.3 延長・撤廃の仕組み

| 種別 | 効果 | 達成難易度 |
|------|------|-----------|
| 延長フラグ | +半年（+15,768,000ターン）※1回のみ | 中盤目標として機能（1周目で達成可能だが困難） |
| 撤廃フラグ | ターン制限の完全撤廃 | 周回前提（複数周回の知識蓄積が必要） |

```
延長判定フロー:
  経過ターン > 31,536,000?
    → 延長フラグ達成済み?
      → Yes: 経過ターン > 47,304,000（1.5年）でゲームオーバー
      → No: ゲームオーバー
    → 撤廃フラグ達成済み?
      → Yes: 制限なし（無制限プレイ）
```

### 11.1 満腹度段階別行動コスト加算

| 段階 | 範囲 | 行動コスト加算 | 追加効果 |
|------|------|---------------|---------|
| 吐き気（Nausea） | 120以上 | +3 | 30%確率行動不可 |
| 過食（Overeating） | 100-119 | +2 | — |
| 満腹（Full） | 80-99 | +1 | — |
| 通常（Normal） | 50-79 | +0 | — |
| 空腹・小（SlightlyHungry） | 40-49 | 30%確率+1 | — |
| 空腹・大（VeryHungry） | 0-39 | +1 | — |
| 飢餓（Starving） | -1～-8 | +2 | 毎ターン1ダメージ |
| 餓死寸前（NearStarvation） | -9 | 移動・攻撃不能 | 毎ターン10ダメージ |
| 餓死（Starvation） | -10 | — | 即死 |
