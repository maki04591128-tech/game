# ターン消費システム設計書

> **📌 実装状況（現在のコードとの整合性）**
>
> | 項目 | 設計 | 実装状況 |
> |------|------|----------|
> | 基本時間単位 | 1ターン = 1秒 | ✅ `TimeConstants.TurnsPerSecond = 1` |
> | 時間定数 | 60ターン=1分、3600=1時間 | ✅ `TimeConstants` で定義済み |
> | ターン消費定数 | 行動別に細かく定義 | ✅ `TurnCosts` クラスで全値定義済み |
> | 状況別移動倍率 | 戦闘×10、隠密×10等 | ⚠️ 定数定義済みだが GameController では現在全行動1ターン消費で簡略化 |
> | TurnAction構造体 | CalculateFinalCost() | ⚠️ 設計段階。現在の実装は GameAction 列挙型 + GameController.ProcessInput() で処理 |
> | 満腹度減少 | 600ターン（10分） | ✅ `TimeConstants.HungerDecayInterval = 600` |
> | 状態異常ティック | 10ターン | ✅ `TimeConstants.StatusEffectTickInterval = 10` |
> | GameTime暦 | 日本語暦12ヶ月30日 | ✅ `GameTime.cs` で実装済み（60ターン=1分） |
>
> **注**: 設計書の行動別ターン消費（攻撃3ターン、食事10ターン等）は `TurnCosts` クラスに定数として定義済みですが、
> 現在の `GameController` ではゲームプレイのテンポを優先して全行動1ターン消費で動作しています。
> 今後の Phase 5 以降で段階的に行動別ターン消費を接続する予定です。

## 1. システム概要

### 1.1 コンセプト

本作のターンシステムは「1ターン = 1秒」を基本単位とし、行動の種類や状況によって消費ターンが変動する。
これにより、リアルな時間経過とゲームプレイのテンポを両立させる。

```
┌─────────────────────────────────────────────────────────────┐
│                    ターン消費システム概念図                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1ターン = 1秒（ゲーム内時間）                               │
│                                                             │
│   【状況別ターン消費】                                        │
│                                                             │
│   通常移動（1マス）        : 1ターン  = 1秒                   │
│   戦闘中/隠密中移動（1マス）: 10ターン = 10秒                  │
│   攻撃（通常攻撃）         : 3ターン  = 3秒                   │
│   スキル使用              : 各スキル定義 = 可変               │
│   魔法詠唱                : 語数依存 = 5~100秒               │
│                                                             │
│   これにより:                                                │
│   ・戦闘は緊張感のある「遅い」時間になる                      │
│   ・探索時は軽快に移動できる                                 │
│   ・時間制限イベントでの戦略性が増す                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 設計思想

| 原則 | 説明 |
|------|------|
| リアリティ | 戦闘中の慎重な動きは時間がかかる |
| 戦略性 | 行動選択にターン消費を考慮させる |
| テンポ | 非戦闘時は快適に移動できる |
| 時間管理 | 限られた時間でのリソース管理 |

## 3. 状況遷移

### 3.1 戦闘状態への移行

```
[通常状態]
    │
    ├─ 敵を視認 → [警戒状態] (Alert)
    │               │
    │               ├─ 敵が接近 → [戦闘状態] (Combat)
    │               │
    │               └─ 敵を見失う(30ターン) → [通常状態]
    │
    └─ 敵から攻撃を受ける → [戦闘状態] (Combat)

[戦闘状態]
    │
    ├─ 敵を全滅させる → [警戒状態] (30ターン) → [通常状態]
    │
    └─ 一定距離逃走 → [追跡状態] (Pursuit) → [警戒状態] → [通常状態]
```

### 3.2 隠密状態

```
[通常状態]
    │
    └─ 隠密スキル発動 → [隠密状態] (Stealth)
                          │
                          ├─ 敵の視界に入る → 発見判定
                          │   ├─ 成功 → [隠密維持]
                          │   └─ 失敗 → [戦闘状態]
                          │
                          ├─ 攻撃する → [戦闘状態] (奇襲ボーナス)
                          │
                          └─ SP切れ/解除 → [通常状態]
```

| 敵タイプ | 移動（戦闘中） | 攻撃 | 特徴 |
|----------|----------------|------|------|
| ゴブリン（俊敏） | 7ターン | 2ターン | 素早い |
| スケルトン（標準） | 10ターン | 3ターン | 普通 |
| オーガ（鈍重） | 15ターン | 5ターン | 遅いが強い |
| スライム（超遅） | 20ターン | 4ターン | 非常に遅い |
| ワイバーン（飛行） | 5ターン | 4ターン | 飛行で速い |

### 5.2 行動順決定

```
行動順 = 前回行動ターン + 今回行動消費ターン

例: 
  プレイヤー: ターン0で攻撃(3ターン消費) → 次はターン3
  ゴブリン: ターン0で攻撃(2ターン消費) → 次はターン2

  ターン2: ゴブリンが先に行動
  ターン3: プレイヤーが行動
```

## 8. 実装仕様

> **📌 現在の実装**: `TurnAction` 構造体は設計段階のもの。実装では `GameAction` 列挙型 + `GameController.ProcessInput()` で行動を処理している。
> ターン消費定数は `TurnCosts` クラスに全て定義済みだが、実際の消費処理はまだ接続されていない（全行動1ターン）。

### 8.1 TurnAction構造体（設計）

```csharp
// ※ 以下は将来の完全版ターンシステム設計。
// 現在の実装では GameAction 列挙型で行動種別を管理し、
// GameController.ProcessInput() で各行動を1ターンとして処理している。
public struct TurnAction
{
    public TurnActionType Type { get; init; }
    public int BaseTurnCost { get; init; }
    public Direction? Direction { get; init; }
    public Character? Target { get; init; }
    public string? SkillId { get; init; }
    public string? Incantation { get; init; }

    // 最終ターン消費計算
    public int CalculateFinalCost(Character actor, GameState state)
    {
        float modifier = 1.0f;

        // 状況修正
        if (Type == TurnActionType.Move)
        {
            modifier *= state.GetMovementModifier(actor);
        }

        // 装備修正
        modifier *= actor.Equipment.GetTurnCostModifier(Type);

        // 状態異常修正
        modifier *= actor.GetStatusEffectModifier(Type);

        // 環境修正
        modifier *= state.CurrentMap.GetEnvironmentModifier(actor.Position, Type);

        return Math.Max(1, (int)(BaseTurnCost * modifier));
    }
}
```

## 10. ゲーム全体のターン制限

### 10.1 基本制限

| 項目 | 値 |
|------|-----|
| 制限ターン数 | 31,536,000ターン（1年） |
| ゲーム内換算 | 365日（12ヶ月 × 30日 + 5日） |
| リセット条件 | 死に戻り発動時に1日目へリセット |
| 制限超過時 | 素性ごとの世界的危機が成就しゲームオーバー |

### 10.2 素性別の制限背景

ターン制限は素性ごとに異なる「世界的危機」として物語に組み込まれている。

| 素性 | 危機の種類 | 概要 |
|------|-----------|------|
| 放浪者 | ダンジョンのスタンピード | 魔物が地上へ溢れ出す |
| 復讐者 | 仇敵の逃亡 | 仇敵が国外へ逃亡する |
| 学者 | 古代遺跡の崩壊 | 手がかりが永久に失われる |
| 亡命者 | 追手の包囲網完成 | 捕縛・処刑される |
| 聖職者 | 裂け目の拡大 | 異界に世界が呑まれる |
| 商人 | 経済崩壊 | 全領地の経済が破綻する |
| 密偵 | クーデター決行 | 王国が滅ぶ |
| 罪人 | 処刑執行 | 猶予期間の終了 |
| 傭兵 | 大戦争の勃発 | 領地間の全面戦争 |
| 預言者 | 終末の預言成就 | 世界の終末 |

> **詳細**: 各素性の危機背景・延長フラグ・撤廃条件は `docs/企画設計書/18_設定資料.md` セクション7.4 を参照

### 10.3 延長・撤廃の仕組み

| 種別 | 効果 | 達成難易度 |
|------|------|-----------|
| 延長フラグ | +半年（+15,768,000ターン）※1回のみ | 中盤目標として機能（1周目で達成可能だが困難） |
| 撤廃フラグ | ターン制限の完全撤廃 | 周回前提（複数周回の知識蓄積が必要） |

```
延長判定フロー:
  経過ターン > 31,536,000?
    → 延長フラグ達成済み?
      → Yes: 経過ターン > 47,304,000（1.5年）でゲームオーバー
      → No: ゲームオーバー
    → 撤廃フラグ達成済み?
      → Yes: 制限なし（無制限プレイ）
```
