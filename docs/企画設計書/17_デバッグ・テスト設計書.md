## 1. テスト戦略

### 1.1 テストピラミッド

```
         /‾‾‾‾\
        / GUI   \        ← GUIオートメーション（FlaUI）: 6メソッド（約108検証項目）
       /‾‾‾‾‾‾‾‾‾\
      / システム検証 \      ← GUIシステム検証テスト: 5メソッド（約121検証項目）
     /‾‾‾‾‾‾‾‾‾‾‾‾‾‾\
    / 統合テスト        \   ← GameController統合テスト + GameTime + 斜め移動: 155件
   /‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾\
  / ユニットテスト          \ ← Core/Engine個別テスト: 7,107件
 /‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾\
```

### 1.2 テスト方針

| 方針 | 説明 |
|------|------|
| 全テスト必須 | コード変更のたびに全テストを実行 |
| 乱数固定 | `TestRandomProvider(fixedValue)` で再現可能なテスト |
| GUIテスト除外 | 通常テスト実行時は `--filter "FullyQualifiedName!~GuiAutomationTests&FullyQualifiedName!~GuiSystemVerificationTests"` |
| 日本語対応 | テスト実行前に UTF-8 エンコーディングを設定 |

## 7. 解決済みバグ一覧

| ID | 概要 | 原因 | 修正内容 |

| BUG-004 | Player.Hp参照エラー | Player.Hpは存在しない | Player.CurrentHp（Character基底クラス）に修正 |
| BUG-005 | FlaUIテストでCanvas要素が取得不可 | WPF CanvasはUI Automationツリーに公開されない | TextBlock等のAutomationId付き要素で検証 |
