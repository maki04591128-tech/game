## 5. テストパターン

### 5.1 乱数固定テスト

```csharp
// TestRandomProvider で乱数を固定し再現可能にする
private readonly TestRandomProvider _fixedRandom = new(0.5);
```

### 5.2 GameController統合テスト

```csharp
// GameControllerを初期化して統合的にテスト
private GameController CreateInitializedController()
{
    var controller = new GameController();
    controller.Initialize();
    return controller;
}
```

### 5.3 FlaUI GUIオートメーションテスト

```csharp
// FlaUI でアプリを起動し、UI Automation で要素を取得・検証
using var app = Application.Launch(AppPath);
using var automation = new UIA3Automation();
var window = app.GetMainWindow(automation, TimeSpan.FromSeconds(10));
// TextBlock等の要素を AutomationId や Name で検索
var statusBar = window.FindFirstDescendant(cf => cf.ByAutomationId("StatusBarText"));
```

### 5.4 命名規則

```
[メソッド名]_[条件]_[期待結果]

例:
GetAffinityMultiplier_ReturnsCorrectValue
Initialize_CreatesPlayerAndMap
ProcessInput_MoveToWall_StaysInPlace
TakeDamage_ReducesHP_TriggersEvent
```

### 6.4 既知のデバッグ注意点

| 項目 | 注意点 |
|------|--------|
| 日本語文字化け | テスト/コマンド実行前にUTF-8設定が必要 |
| FlaUIテスト | ウィンドウが実際に起動するため、CI環境では除外 |
| 乱数依存 | マップ生成・戦闘は乱数に依存。テストでは固定乱数を使用 |
| デバッグマップ敵配置 | 敵は右上隅(26-28, 3-7)に配置。プレイヤー中央(16,12)から遠く、テスト中の意図しない戦闘を防止 |
| 空腹度テスト | HungerDecayInterval=600ターン。大きいマップではレンダリング負荷で待ち時間のマージンを広めに取る必要あり |

## 7. 解決済みバグ一覧

| ID | 概要 | 原因 | 修正内容 |
|----|------|------|---------|
| BUG-001 | Food.Use()が満腹度を減少させていた | ModifyHunger の引数符号が逆 | +nutrition に修正 |
| BUG-002 | StatusEffect名前空間エラー | Entities名前空間にStatusEffectが定義されていた | Core名前空間に移動 |
| BUG-003 | コンソール日本語文字化け | デフォルトエンコーディングが Shift_JIS | UTF-8 P/Invoke設定で対応 |
| BUG-004 | Player.Hp参照エラー | Player.Hpは存在しない | Player.CurrentHp（Character基底クラス）に修正 |
| BUG-005 | FlaUIテストでCanvas要素が取得不可 | WPF CanvasはUI Automationツリーに公開されない | TextBlock等のAutomationId付き要素で検証 |
