## 1. テスト戦略

### 1.1 テストピラミッド

```
         /‾‾‾‾\
        / GUI   \        ← GUIオートメーション（FlaUI）: 少数
       /‾‾‾‾‾‾‾‾‾\
      / 統合テスト   \     ← GameController統合テスト: 中程度
     /‾‾‾‾‾‾‾‾‾‾‾‾‾‾\
    / ユニットテスト    \   ← Core/Engine個別テスト: 多数
   /‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾\
```

### 1.2 テスト方針

| 方針 | 説明 |
|------|------|
| 全テスト必須 | コード変更のたびに全テストを実行 |
| 乱数固定 | `TestRandomProvider(fixedValue)` で再現可能なテスト |
| GUIテスト除外 | 通常テスト実行時は `--filter "FullyQualifiedName!~GuiAutomationTests"` |
| 日本語対応 | テスト実行前に UTF-8 エンコーディングを設定 |

## 5. テストパターン

### 5.1 乱数固定テスト

```csharp
// TestRandomProvider で乱数を固定し再現可能にする
private readonly TestRandomProvider _fixedRandom = new(0.5);
```

### 5.2 GameController統合テスト

```csharp
// GameControllerを初期化して統合的にテスト
private GameController CreateInitializedController()
{
    var controller = new GameController();
    controller.Initialize();
    return controller;
}
```

### 5.3 命名規則

```
[メソッド名]_[条件]_[期待結果]

例:
GetAffinityMultiplier_ReturnsCorrectValue
Initialize_CreatesPlayerAndMap
ProcessInput_MoveToWall_StaysInPlace
TakeDamage_ReducesHP_TriggersEvent
```

## 6. デバッグ手法

### 6.1 コンソールデモ（InteractiveGameDemo）

| 項目 | 説明 |
|------|------|
| ファイル | `src/RougelikeGame.App/InteractiveGameDemo.cs` |
| 用途 | GUIなしでゲームロジックをテスト |
| 起動 | `dotnet run --project src/RougelikeGame.App` |
| 機能 | コンソール上でマップ表示、移動、攻撃、ステータス確認 |

### 6.2 Visual Studio デバッグ

| 手法 | 用途 |
|------|------|
| ブレークポイント | 特定処理のステップ実行 |
| ウォッチウィンドウ | Player/Enemy/Map 状態の確認 |
| 出力ウィンドウ | ビルドログ、テスト結果の確認 |
| 診断ツール | メモリ/CPU使用量のモニタリング |

### 6.3 既知のデバッグ注意点

| 項目 | 注意点 |
|------|--------|
| 日本語文字化け | テスト/コマンド実行前にUTF-8設定が必要 |
| FlaUIテスト | ウィンドウが実際に起動するため、CI環境では除外 |
| 乱数依存 | マップ生成・戦闘は乱数に依存。テストでは固定乱数を使用 |

## 7. 解決済みバグ一覧

| ID | 概要 | 原因 | 修正内容 |
