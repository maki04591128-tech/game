## 1. エンチャントシステム

### 1.1 概要

装備品に魔法効果を付与するシステム。TES V: Skyrimの付呪システムを参考に、
既存のCraftingSystem.EnchantWeapon（属性エンチャント）を大幅に拡張する。

### 1.2 参考作品

- **TES V: Skyrim**: 付呪台→魂石消費→効果選択→装備に付与
- **Diablo**: ランダムエンチャント、接頭辞/接尾辞システム

### 1.3 設計方針

| 要素 | 内容 |
|------|------|
| 魂石システム | 敵撃破時に「魂石」を入手。敵のランク/種族で魂石の品質が変動 |
| 効果抽出 | 特殊装備を分解して効果を学習（一度学習すれば何度でも付与可能） |
| 付呪台 | 街の鍛冶屋施設 or ダンジョン内の特殊部屋で使用 |
| 効果スロット | 装備品ごとに1〜3スロット（品質依存） |
| 付呪効果例 | 火炎付与/氷結付与/吸血/麻痺付与/経験値増加/ドロップ率上昇 等 |
| 魂石品質 | 欠片(Common)→小(Elite)→中(Rare)→大(Boss)→極大(HiddenBoss) |

### 1.4 既存システムとの関連

- `CraftingSystem.EnchantWeapon()`: 現在は属性付与のみ。付呪効果リストに拡張
- `Equipment.Enhancement`: 強化値(+1〜+10)とは別枠で管理
- `Element` enum: 属性系エンチャントの基盤として利用
- `EnemyRank`: 魂石品質の決定基準

### 1.5 対応バージョン

**Ver.prt.0.2**（システム実装）

## 2. 敵種族分類システム

### 2.1 概要

敵を生物学的な種族で分類し、種族ごとに異なる特性・弱点・行動パターンを持たせる。

### 2.2 設計方針

| 種族 | 特徴 | 弱点 | 耐性 | 代表的な敵 |
|------|------|------|------|-----------|
| 獣 (Beast) | 高AGI、群れ行動、夜行性 | 火 | 物理耐性低 | Wolf, Giant Spider, Forest Wolf |
| 人型 (Humanoid) | 装備品使用、戦術的AI | 種族依存 | なし | Goblin, Dark Elf, Bandit |
| 不定形 (Amorphous) | 物理耐性高、分裂能力 | 魔法 | 斬撃/刺突耐性 | Slime, Ooze |
| 不死 (Undead) | 毒/睡眠無効、暗闘 | 光/聖 | 毒/闇耐性 | Skeleton, Draugr, Ghost |
| 悪魔 (Demon) | 高MND、魔法使用 | 聖 | 闇/呪い耐性 | Demon, Devil |
| 竜 (Dragon) | 全ステ高、ブレス攻撃 | 種族依存 | 属性ブレス耐性 | Dragon, Wyrm |
| 植物 (Plant) | 再生能力、状態異常付与 | 火/氷 | 地/水耐性 | Treant, Mandragora |
| 昆虫 (Insect) | 高速、毒攻撃、群れ | 火 | 精神攻撃無効 | Giant Ant, Wasp |
| 精霊 (Spirit) | 物理攻撃半減、属性攻撃 | 対属性 | 物理耐性 | Fire Spirit, Ice Spirit |
| 構造体 (Construct) | 状態異常完全無効 | 雷 | 毒/精神無効 | Stone Golem, Iron Golem |

### 2.3 既存システムとの関連

- `EnemyType` enum: 行動パターン（AI）。種族とは別軸で管理
- `EnemyRank` enum: 強さランク。種族とは別軸
- `EnemyDefinition`: `MonsterRace` プロパティを追加
- `CombatSystem`: 種族弱点/耐性を戦闘計算に組み込み

### 2.4 対応バージョン

**Ver.prt.0.2**（システム実装）

## 3. 拡張ドロップテーブル

### 3.1 概要

敵の種族・出現マップ・ランク・状況に応じてドロップテーブルを細分化する。
現在のDropTableSystemの14テーブルを大幅に拡張。

### 3.2 設計方針

| 要因 | 影響 |
|------|------|
| 敵種族 | 獣→素材/食料、人型→装備/金貨、不定形→特殊素材 |
| 出現マップ | 森林→薬草/木材、山岳→鉱石/宝石、海岸→魚/真珠 |
| 敵ランク | Common→低品質、Boss→高品質/ユニーク |
| 時間帯 | 夜間→レアドロップ率微増 |
| 季節 | 季節限定素材（季節システム連動） |

### 3.3 既存システムとの関連

- `DropTableSystem`: 基盤として拡張
- `EnemyDefinition.DropTableId`: 種族別テーブルIDに対応
- `TerritoryId`: マップ別テーブルの切り替え条件

### 3.4 対応バージョン

**Ver.prt.0.2**（システム実装）

## 4. スキルツリーシステム

### 4.1 概要

Path of ExileやGrim Dawnを参考にした大規模スキルツリー。
既存の10職業別スキルツリーをベースに、ノード接続・パッシブスキル・
クラス横断パスを追加する。

### 4.2 参考作品

- **Path of Exile**: 巨大共有パッシブツリー、クラス開始位置が異なる
- **Grim Dawn**: デュアルクラス、星座システム（追加パッシブツリー）
- **Diablo II**: スキルポイント振り分け、シナジー

### 4.3 設計方針

| 要素 | 内容 |
|------|------|
| ツリー構造 | 職業別メインツリー＋共有パッシブツリー |
| ノード種類 | アクティブスキル/パッシブ強化/ステータスノード/キーストーン |
| スキルポイント | レベルアップ時に獲得（1pt/Lv） |
| キーストーン | 強力だがデメリット付きの特殊パッシブ（例: 「ガラスの大砲」攻撃2倍/防御半減） |
| リスペック | 街の施設で有料リスペック（ゴールド消費） |
| クラス横断 | 隣接クラスのツリーに一定範囲アクセス可能 |

### 4.4 既存システムとの関連

- `SkillSystem`: 50+スキル、10ツリー。ノード接続グラフに拡張
- `SkillDefinition`: Prerequisite（前提スキル）プロパティは実装済み
- `Player.SkillPoints`: レベルアップ時のポイント管理

### 4.5 対応バージョン

**Ver.prt.0.2**（システム実装）

## 5. 熟練度システム

### 5.1 概要

各行動を繰り返すことで熟練度が上昇し、その行動の効果が向上するシステム。
KenshiやElonaのスキル経験値システムを参考にする。

### 5.2 参考作品

- **Kenshi**: 戦闘/労働/移動等あらゆる行動にスキル経験値
- **Elona**: 行動ごとにスキルレベルが上昇、使わないと低下
- **TES V: Skyrim**: 行動ベースのスキルレベリング

### 5.3 設計方針

| 熟練度カテゴリ | 対象行動 | 効果 |
|---------------|---------|------|
| 剣術 | 剣系武器で攻撃 | 命中率↑、ダメージ↑ |
| 槍術 | 槍系武器で攻撃 | 命中率↑、クリティカル率↑ |
| 弓術 | 弓で射撃 | 射程↑、命中率↑ |
| 格闘 | 素手攻撃 | ダメージ↑、カウンター率↑ |
| 魔術 | 魔法詠唱 | MP効率↑、成功率↑ |
| 鍛冶 | 合成・強化 | 成功率↑、品質↑ |
| 採掘 | 素材採取 | 入手量↑、レア率↑ |
| 交渉 | 商売・会話 | 値引率↑、選択肢増加 |
| 隠密 | 罠回避・隠れる | 発見率↑、奇襲率↑ |
| 探索 | 隠しドア/罠発見 | 発見距離↑、自動発見 |
| 錬金 | 薬品作成 | 効果↑、副作用↓ |
| 信仰 | 祈り・宗教行動 | 信仰度上昇量↑ |

### 5.4 レベリング設計

- 行動実行時に経験値獲得（行動難易度に比例）
- 一定経験値で熟練度レベルアップ（必要経験値は指数増加）
- 使用しない熟練度は緩やかに減少（Elona方式、最低値保証あり）
- 既存の `ClassEquipmentSystem` の適性倍率と乗算で最終効果を算出

### 5.5 既存システムとの関連

- `Player.WordMastery`: 魔法語彙の熟練度として部分実装済み
- `ClassEquipmentSystem`: 職業別装備適性（13カテゴリ）との連携
- `CombatSystem`: 熟練度ボーナスをダメージ/命中計算に組み込み

### 5.6 対応バージョン

**Ver.prt.0.2**（システム実装）

## 6. 能力値フラグシステム

### 6.1 概要

プレイヤーの能力値が一定の閾値を超えた場合にフラグが変動し、
新たなイベント・選択肢・NPCの反応が解放されるシステム。

### 6.2 設計方針

| 能力値 | 閾値 | フラグ例 |
|--------|------|---------|
| STR ≥ 25 | 高 | 「怪力」: 岩の障害物を破壊可能、力自慢NPC挑戦イベント |
| INT ≥ 25 | 高 | 「博識」: 古文書解読、賢者NPC専用会話 |
| PER ≥ 25 | 高 | 「鷹の目」: 隠しドア自動発見、遠距離の敵察知 |
| AGI ≥ 25 | 高 | 「韋駄天」: 逃走確率100%、特殊ルート解放 |
| CHA ≥ 20 | 中 | 「魅力的」: ショップ追加値引、NPC好感度初期値↑ |
| LUK ≥ 20 | 中 | 「強運」: レアドロップ率↑、カジノ特殊イベント |

### 6.3 既存システムとの関連

- `Player.EffectiveStats`: 能力値の取得元
- `DialogueSystem.ConditionFlag`: 条件フラグとして連携
- `BackgroundClearSystem`: フラグ追跡（bool/int）の基盤として利用可能

### 6.4 対応バージョン

**Ver.prt.0.2**（システム実装）、**Ver.α**（テキストコンテンツ）

## 7. フラグ選択肢変動システム

### 7.1 概要

特定のフラグ（能力値フラグ、カルマ、宗教、種族、クエスト進行等）に応じて
会話やイベントの選択肢が変化するシステム。

### 7.2 設計方針

既存の `DialogueChoice.RequiredFlag` を拡張し、複数条件の AND/OR 判定、
数値比較（≥, ≤）、否定条件（!flag）を追加する。

| 条件種別 | 例 |
|---------|-----|
| フラグ存在 | `has:quest_dragon_slain` |
| フラグ値比較 | `karma >= 50` |
| 能力値条件 | `stat:STR >= 20` |
| 種族条件 | `race:Elf` |
| 宗教条件 | `religion:LightTemple` |
| 熟練度条件 | `mastery:sword >= 10` |
| 複合条件 | `has:guild_registered AND karma >= 30` |

### 7.3 既存システムとの関連

- `DialogueSystem`: `RequiredFlag` プロパティ（現在は単一フラグ名のみ）
- `DialogueChoice`: 条件判定メソッドの拡張
- `BackgroundClearSystem`: フラグストレージの共有

### 7.4 対応バージョン

**Ver.prt.0.2**（システム実装）、**Ver.α**（分岐テキスト執筆）

## 8. スキル合成システム

### 8.1 概要

異なる職業のスキルを組み合わせて新しいスキルを生成するシステム。

### 8.2 設計方針

| 素材スキル1 | 素材スキル2 | 合成スキル | 効果 |
|------------|------------|-----------|------|
| 強撃（Fighter） | ファイアボール（Mage） | 炎撃 | 物理+火属性の複合攻撃 |
| ヒール（Cleric） | 毒霧（Necromancer） | 浄化波 | 範囲回復+状態異常解除 |
| 隠密（Thief） | バックスタブ（Thief） | 暗殺 | 確定クリティカル+即死判定 |
| 盾打ち（Knight） | 挑発（Fighter） | 鉄壁の守護 | 味方全体のダメージ軽減 |

### 8.3 条件

- 両方のスキルの熟練度が一定以上
- 合成用の特殊アイテム「スキルの書」を消費
- 合成後も元のスキルは使用可能

### 8.4 既存システムとの関連

- `SkillSystem`: 50+スキル、カテゴリ別管理
- `SkillDefinition`: 合成レシピの定義追加
- `CraftingSystem`: 合成UIの共用可能性

### 8.5 対応バージョン

**Ver.prt.0.2**（システム実装）

## 9. 敵AI拡張（種族・状態対応）

### 9.1 概要

敵の種族や現在の状態に応じてAIの行動パターンが動的に変化するシステム。

### 9.2 設計方針

| 条件 | AI変化 |
|------|--------|
| 獣種族（Beast） | 群れ行動: 仲間が近くにいると連携攻撃、1体になると逃走 |
| 不死種族（Undead） | 恐怖無効、HP低下でも逃走しない、暗所で強化 |
| 人型種族（Humanoid） | アイテム使用、罠設置、仲間呼び |
| HP低下（<30%） | 回復アイテム使用優先、逃走判定強化 |
| 状態異常中 | 混乱→ランダム移動、恐怖→逃走固定、狂気→味方攻撃あり |
| 夜間 | 夜行性種族は全ステータス+10%、昼行性は-10% |
| ボス戦フェーズ | HP50%以下で行動パターン変化（第2形態） |

### 9.3 既存システムとの関連

- `IAIBehavior`: 5種（Normal/Aggressive/Defensive/Coward/Ambusher/Pack/Boss）
- `RangedBehavior`/`BerserkerBehavior`/`SummonerBehavior`: 特殊AI
- `StatusEffect`: 状態異常による行動制限（Charm, Madness等は実装済み）

### 9.4 対応バージョン

**Ver.prt.0.2**（システム実装）

## 10. カルマシステム

### 10.1 概要

善行や犯罪行為でカルマ値が変動し、NPCの反応・ショップ価格・
イベント展開・エンディング分岐に影響するシステム。

### 10.2 参考作品

- **Fallout**: カルマ値による評判変動、NPCの態度変化
- **Ultima**: 8つの徳（誠実/勇気/慈悲等）

### 10.3 設計方針

| カルマ範囲 | 称号 | 影響 |
|-----------|------|------|
| 80〜100 | 聖人 | ショップ大幅値引、光の神殿恩恵強化、闇の勢力から敵視 |
| 50〜79 | 善人 | NPC好意的、一部クエスト解放 |
| 20〜49 | 普通 | 標準 |
| -19〜19 | 中立 | 影響なし |
| -49〜-20 | 悪漢 | 一部NPC警戒、闇市アクセス解放 |
| -79〜-50 | 悪党 | ショップ割増、一部街で入場制限 |
| -100〜-80 | 外道 | 大半のNPCが敵対、暗黒系クエスト解放 |

### 10.4 カルマ変動条件

| 行動 | カルマ変動 |
|------|-----------|
| クエスト達成（善行系） | +5〜+15 |
| NPC救助 | +10 |
| 敵対NPCでない者への攻撃 | -20 |
| 窃盗 | -10 |
| 墓荒らし | -5 |
| 貧しいNPCへの施し | +5 |
| 闇市での取引 | -3 |

### 10.5 既存システムとの関連

- `ReligionSystem`: 宗教的な善悪判定と連携（光の神殿=善寄り、暗黒の教団=悪寄り）
- `NpcSystem.Affinity`: NPC好感度にカルマ補正を加算
- `ShopSystem`: 価格倍率にカルマ補正を乗算
- `DialogueSystem`: カルマ条件による選択肢分岐

### 10.6 対応バージョン

**Ver.prt.0.2**（システム実装）、**Ver.α**（カルマ反応テキスト）

## 11. 仲間・傭兵システム

### 11.1 概要

NPCを仲間としてパーティに加え、共にダンジョンを冒険できるシステム。

### 11.2 参考作品

- **Elona**: ペット/仲間の自律AI、装備変更、育成
- **Dungeon Crawl**: 同行者システム、AI指示

### 11.3 設計方針

| 要素 | 内容 |
|------|------|

## 25. 攻撃属性システム

### 25.1 概要

既存のElement enum（Fire/Lightning/Ice/Water/Earth/Wind/Light/Dark/Poison/Holy/Curse/None の12種）を
物理攻撃・魔法攻撃の両方に統合的に適用するシステム。
現状は魔法スクロールやエンチャントで部分的に使用されているが、
通常攻撃・スキル攻撃にも属性を付与し、敵の属性耐性との相互作用を体系化する。

### 25.2 参考作品

- **ポケットモンスター**: タイプ相性テーブル（効果抜群/いまひとつ/無効）
- **Final Fantasy**: 属性弱点/吸収/無効の3段階耐性
- **Dark Souls**: 物理/魔法/炎/雷/闇の属性ダメージ分離計算

### 25.3 設計方針

| 要素 | 内容 |
|------|------|
| 属性源 | 武器固有属性、エンチャント付与、スキル属性、バフ効果 |
| 耐性段階 | 弱点(×1.5) / 通常(×1.0) / 耐性(×0.5) / 無効(×0) / 吸収(回復) |
| 相性テーブル | Element × EnemyRace/EnemyType のマトリクス |
| 複合属性 | 武器の物理属性(Slash/Pierce/Blunt)＋元素属性の二重計算 |
| UI表示 | ダメージ数値に属性アイコン表示、弱点ヒット時のエフェクト |

### 25.4 既存システムとの関連

- `Element` enum（Enums.cs）: 12属性定義済み
- `AttackType` enum: Slash/Pierce/Blunt の物理種別と並列管理
- `CraftingSystem.EnchantWeapon()`: 属性エンチャント処理
- `SkillSystem`: スキル使用時の属性適用
- `CombatSystem`: ダメージ計算に属性倍率を統合

### 25.5 対応バージョン

**Ver.prt.0.2**（属性相性テーブル・ダメージ計算統合）

## 26. アイテム等級システム

### 26.1 概要

アイテムの品質（等級）によりステータス効果が変動するシステム。
既存のItemRarity（Common〜Unique の6段階）とは別軸で、
同一アイテムでも製造品質の違いによる性能差を表現する。

### 26.2 参考作品

- **Mount & Blade**: 武器・防具の品質段階（粗悪/普通/上質/精巧/傑作）
- **Kenshi**: 装備品質によるステータス係数変動
- **Valheim**: 素材品質による完成品の段階差

### 26.3 設計方針

| 等級 | 名称 | ステータス係数 | 価格係数 | 出現率 |
|------|------|-------------|---------|--------|
| 1 | 粗悪品 | ×0.7 | ×0.4 | 20% |
| 2 | 廉価品 | ×0.85 | ×0.7 | 25% |
| 3 | 標準品 | ×1.0 | ×1.0 | 30% |
| 4 | 良品 | ×1.15 | ×1.5 | 15% |
| 5 | 上質品 | ×1.3 | ×2.5 | 8% |
| 6 | 傑作品 | ×1.5 | ×5.0 | 2% |

- 鍛冶熟練度が高いほど上位等級の作成確率上昇
- ショップ商品は領地の発展度に応じた等級分布
- 等級はアイテム名に接頭辞として表示（例: 「粗悪な鉄の剣」「上質な鉄の剣」）

### 26.4 既存システムとの関連

- `ItemRarity` enum（Item.cs）: レアリティとは別軸の品質管理
- `IItem.BasePrice`: 等級係数を乗算して実効価格を算出
- `SmithingSystem`: 鍛冶作成時に熟練度依存で等級決定
- `DropTableSystem`: 敵ランク/マップ難度で等級分布を変動

### 26.5 対応バージョン

**Ver.prt.0.2**（等級enum追加・ステータス係数計算・鍛冶連動）

### 27.3 設計方針

| 施設種類 | 機能 | 必要素材 |

### 27.5 対応バージョン

### 28.1 概要

フィールドやダンジョン内の採取ポイントから素材を収集するシステム。
既存のMaterialDeposit（RandomEventType）イベントを体系化・拡張する。

### 28.2 参考作品

- **モンスターハンター**: フィールド内の採掘/採取/虫取りポイント

- **Valheim**: 木の伐採、岩の採掘、ベリー収穫

### 28.3 設計方針

| 採取種別 | 対象 | 必要装備/スキル | 取得素材例 |
|---------|------|---------------|-----------|
| 採掘 | 鉱脈/岩 | つるはし/採掘熟練度 | 鉄鉱石、宝石原石、古代の欠片 |

### 28.4 既存システムとの関連

## 30. 剥ぎ取りシステム

### 30.1 概要

敵を倒した際にアイテムが自動ドロップするのではなく、死体をインタラクトして「剥ぎ取り」で
アイテムを取得する方式。よりリアルなルーティング体験を提供する。

### 30.2 参考作品

- **モンスターハンター**: 倒した敵から剥ぎ取り（複数回可能）
- **Dark Souls**: 死体からアイテム取得（光るアイテム拾い）
- **Kenshi**: 倒れた敵/味方からのアイテム回収

### 30.3 設計方針

| 要素 | 内容 |
|------|------|
| 死体タイル | 敵死亡時にCorpseタイルとしてマップ上に残存 |
| インタラクト | 死体に隣接して調べる操作→剥ぎ取り画面 |
| 剥ぎ取り回数 | 敵ランク依存（Common:1回、Elite:2回、Boss:3回） |
| 所要時間 | 1回の剥ぎ取り＝1ターン |
| 腐敗 | 時間経過で死体が消滅（一定ターン後に自動消去） |

### 30.4 既存システムとの関連

- `DropTableSystem`: GenerateLoot()を剥ぎ取り時呼び出しに変更

- `TileType`: Corpse タイルの追加
- `GameController`: 死体インタラクト処理の追加
- 敵種族分類（§2）: 種族別素材テーブル

### 30.5 対応バージョン

## 31. 物品価値変動システム

### 31.1 概要

アイテムの売買価格が固定ではなく、領地・商人の種類・需要と供給によって
動的に変動するシステム。交易と商売の戦略性を追加する。

### 31.2 参考作品

- **Mount & Blade**: 街ごとの価格差、交易による利益
- **Kenshi**: 都市間の価格差、密輸品の高額取引
- **Recettear**: 需要と供給による価格変動、値引き交渉

### 31.3 設計方針

| 変動要因 | 効果 | 例 |
|---------|------|-----|
| 領地特産 | 特産品は安く、他領地では高い | 山岳領地の鉄鉱石は安価、南部では高額 |
| 商人種別 | 専門商人は専門品を安く売る | 武器商は武器が安い、雑貨屋は高い |
| 需要変動 | 季節/イベントで需要変化 | 冬は防寒具の需要↑→価格↑ |
| 供給量 | プレイヤーの大量売却で価格下落 | 同一アイテム大量売却→買取価格↓ |
| カルマ影響 | カルマ高→割引、カルマ低→割増 | 悪評のプレイヤーは足元を見られる |
| 評判影響 | 領地評判で価格変動 | 英雄扱いの領地では割引 |

- 基本価格（BasePrice）× 領地係数 × 商人係数 × 需給係数 × カルマ係数 = 最終価格
- 価格情報は「密輸商人」「情報屋」NPCから入手可能

### 31.4 既存システムとの関連

- `IItem.BasePrice`: 基本価格からの変動計算
- `TerritoryId` enum: 領地ごとの価格係数管理
- カルマシステム（§10）: カルマ値による価格変動
- 評判・名声システム（§19.5）: 評判値による価格変動
- `Player.Gold`: ゴールド管理

### 31.5 対応バージョン

**Ver.prt.0.3**（基本変動メカニクス）、**Ver.prt.1.0**（需給シミュレーション）

## 32. 描画範囲外抑制・軽量化

### 32.1 概要

プレイヤーの視界外（描画範囲外）にいる敵やNPCの行動処理を抑制し、
ゲームのパフォーマンスを向上させるシステム。

### 32.2 参考作品

- **Dwarf Fortress**: アクティブ領域とスリープ領域の区分
- **RimWorld**: 画面外のpawnは簡易行動で処理
- **Kenshi**: チャンク単位のアクティブ/非アクティブ切替

### 32.3 設計方針

| 処理レベル | 範囲 | 処理内容 |
|-----------|------|---------|
| アクティブ | 視界内（SightRange以内） | 完全AI行動・アニメーション |
| 準アクティブ | 視界外〜同フロア | 簡易AI（移動のみ、戦闘判定省略） |
| スリープ | 別フロア/別マップ | 行動停止（状態保存のみ） |

- ターン処理時にアクティブ範囲の判定を行い、範囲外エンティティは簡易処理
- フロア移動時に非アクティブ→アクティブへの復帰処理
- 大規模マップ（100×100以上）でのFPS低下を防止
- 敵のSightRange（デフォルト8）/HearingRange（デフォルト5）を活用

### 32.4 既存システムとの関連

- `Enemy.SightRange` / `HearingRange`: アクティブ判定の基準距離
- `Enemy.IsVisible()` / `CanHear()`: 距離判定ロジック
- `GameController`: ターン処理ループの最適化
- `DungeonGenerator`: フロアサイズに応じた閾値設定

### 32.5 対応バージョン

**Ver.prt.0.3**（基本抑制ロジック）、**Ver.prt.1.0**（チューニング・プロファイリング）

## 33. 時刻行動変化システム

### 33.1 概要

ゲーム内時刻（昼/夜/夕方/早朝）によってNPCや敵の行動パターンが変化するシステム。
既存のGameTime（ターン進行による時間管理）と連動する。

### 33.2 参考作品

- **TES V: Skyrim**: NPC日課スケジュール（就寝/仕事/食事）
- **ゼルダの伝説 BoTW**: 昼夜で敵種類・行動変化
- **Kenshi**: 昼夜サイクルによるNPC行動・店営業時間

### 33.3 設計方針

| 時間帯 | 時刻範囲 | 敵の変化 | NPCの変化 |
|--------|---------|---------|----------|
| 早朝 | 5:00-8:00 | 夜行性の敵が弱体化/撤退 | 商人が店を開ける |
| 昼 | 8:00-17:00 | 標準行動 | 通常営業・会話可能 |
| 夕方 | 17:00-20:00 | 一部の敵が活性化 | 商人が店を閉める |
| 夜 | 20:00-5:00 | 夜行性の敵が強化/出現増 | 大半のNPCが就寝 |

- 敵出現テーブルの時間帯別フィルタリング
- NPC営業時間（ショップ開閉）
- 夜間は視界低下（SightRange 減少）
- 一部ダンジョンは時刻影響なし（地下のため）
- GameTime.Hour から時間帯を自動判定

### 33.4 既存システムとの関連

- `GameTime`: Hour プロパティで時刻取得
- `GameConstants`: TurnsPerHour=3600, DayNightCycleHalf=43200
- `Enemy.SightRange`: 夜間の視界低下適用
- `NpcSystem`: NPC行動スケジュールの追加
- `WorldMapSystem`: ロケーション営業時間の管理

### 33.5 対応バージョン

**Ver.prt.0.2**（昼夜基本ロジック）、**Ver.prt.0.3**（NPCスケジュール詳細化）

## 34. シンボルマップランダムイベント拡張

### 34.1 概要

既存のワールドマップ（シンボルマップ）移動時のランダムイベントを大幅に拡張し、
救助・襲撃・交易等の多彩なイベントを追加する。

### 34.2 参考作品

- **Mount & Blade**: ワールドマップ移動中のランダムエンカウント
- **FTL**: セクター間移動時のイベント分岐
- **Darkest Dungeon**: 道中イベント（キャンプ/遭遇/発見）

### 34.3 設計方針

| イベント種別 | 内容 | 発生条件 |
|------------|------|---------|
| 救助イベント | 負傷NPC発見→救助で評判↑/報酬 | 街道上・ランダム |
| 襲撃イベント | 盗賊/モンスターの待ち伏せ | 辺境・夜間 |
| 交易商人 | 移動商人と遭遇→売買 | 街道上・昼間 |
| 天災イベント | 落石/洪水→迂回必要 | 山岳/河川・季節依存 |
| 謎の旅人 | 情報提供/アイテム交換 | ランダム |
| 野営地発見 | 休憩可能・HPリカバリー | 野原/森 |
| 遺跡入口発見 | 隠しダンジョンへのアクセス | 探索/PER依存 |
| 行商キャラバン | 護衛クエスト→報酬 | 街道上 |

- 既存15種のRandomEventTypeに新イベントを追加
- イベント発生率は領地・時間帯・天候・カルマで変動
- 選択肢（救助する/見捨てる等）によるカルマ/評判変動

### 34.4 既存システムとの関連

- `RandomEventType` enum: 既存15種への追加
- `WorldMapSystem`: イベント結果処理メソッド群の拡張
- `TravelEventWindow.xaml.cs`: GUI表示
- カルマシステム（§10）: イベント選択によるカルマ変動
- 時刻行動変化（§33）: 時間帯によるイベント種別フィルタ

### 34.5 対応バージョン

**Ver.prt.0.3**（新イベント追加）、**Ver.α**（イベントテキスト・選択肢）

## 35. ダンジョン特徴別ランダム生成

### 35.1 概要

ダンジョンの種類（洞窟/遺跡/塔/地下水路等）に応じて、ランダム生成のロジックや
出現タイル・敵・アイテム配置を変化させるシステム。
既存のBSP法ダンジョン生成を特徴パラメータで制御する。

### 35.2 参考作品

- **Caves of Qud**: バイオーム特性によるダンジョン変化
- **Cogmind**: フロアテーマによるタイル・敵・アイテム変化
- **Dungeons of Dredmor**: フロアごとのテーマ装飾

### 35.3 設計方針

| ダンジョン特徴 | 部屋形状 | 通路特性 | 特殊タイル | 出現敵傾向 |
|--------------|---------|---------|-----------|-----------|
| 自然洞窟 | 不規則・広い | 曲がりくねった通路 | 水場/溶岩/鍾乳石 | 獣/昆虫/スライム |
| 古代遺跡 | 整然とした大部屋 | 直線通路・列柱 | 石碑/罠/封印扉 | 不死/構造体 |
| 魔導塔 | 小部屋・螺旋 | 縦構造（階段多い） | 魔法陣/結界 | 精霊/悪魔 |
| 地下水路 | 長い通路 | 水路に沿った直線 | 水場/水門 | 水棲/不定形 |
| 鉱山 | 横穴型 | 支柱付き通路 | 鉱脈/崩落 | 構造体/植物 |
| 墓地/カタコンベ | 小部屋・棺桶 | 狭い通路 | 棺桶/祭壇 | 不死/悪魔 |

- 既存のDungeonGenerationParametersに特徴パラメータを追加
- SpecialFloorType（Normal/Shop/BossRoom等）と組み合わせ
- フロア深度に応じて特徴の強度が変化

### 35.4 既存システムとの関連

- `DungeonGenerator`: Generate()メソッドのパラメータ拡張
- `RoomCorridorGenerator`: 部屋形状・通路パターンの多様化
- `FeaturePlacer`: タイル装飾の特徴依存配置
- `SpecialFloorType` enum: 特徴テーマとの組み合わせ
- 敵種族分類（§2）: ダンジョン特徴×敵種族の対応テーブル

### 35.5 対応バージョン

**Ver.prt.0.3**（特徴パラメータ基盤）、**Ver.prt.1.0**（全特徴実装）

## 36. 勢力地変動システム

### 36.1 概要

プレイヤーの敵討伐数や時間経過によって、各領地の勢力バランスが動的に変動するシステム。
特定の敵を大量に討伐すると、その地域の安全度が上がり、逆に放置すると敵勢力が拡大する。

### 36.2 参考作品

- **Shadow of Mordor/War**: ネメシスシステム（敵勢力の動的変化）
- **Mount & Blade**: 領地の支配勢力変動
- **Kenshi**: 勢力関係の動的変化

### 36.3 設計方針

| 要素 | 内容 |
|------|------|
| 勢力パラメータ | 各領地に「安全度」（0-100）を設定 |
| 討伐影響 | 敵討伐→安全度↑（敵ランクに応じた上昇量） |
| 時間経過 | 未訪問領地の安全度は緩やかに低下 |
| 安全度効果 | 高: 敵出現↓/NPC活動↑/交易活発、低: 敵出現↑/NPC避難/物価高騰 |
| ボス討伐 | 特定ボス討伐で安全度が大幅上昇＋新ロケーション開放 |
| イベント連動 | 安全度に応じたランダムイベント変化 |

- TerritoryId（6領地）ごとに安全度を管理
- 討伐カウントは敵種族×領地の組み合わせで記録
- 安全度の変動は季節イベント（冬は魔物活性化等）とも連動

### 36.4 既存システムとの関連

- `TerritoryId` enum: 6領地の管理単位
- `Faction` enum: Player/Enemy/Neutral/Friendly
- `EnemyRank`: Common/Elite/Rare/Boss の討伐ポイント差
- `WorldMapSystem`: 領地状態の表示・イベント変化
- 季節システム（§12）: 季節による勢力変動

### 36.5 対応バージョン

**Ver.prt.0.3**（安全度管理・討伐カウント）、**Ver.prt.1.0**（変動イベント・NPC連動）

## 37. グリッドインベントリ

### 37.1 概要

アイテムがグリッド（方眼紙状のマス目）上に配置されるインベントリシステム。
アイテムごとにサイズ（縦×横マス）が異なり、限られたスペースに効率よく
配置する戦略性を追加する。

### 37.2 参考作品

- **Escape from Tarkov**: グリッドインベントリ（アイテムサイズ管理）
- **Diablo 2**: テトリス型インベントリ
- **Resident Evil 4**: アタッシュケースのグリッド管理

### 37.3 設計方針

| 要素 | 内容 |
|------|------|
| グリッドサイズ | 基本10×6マス（バッグ/鎧で拡張可能） |
| アイテムサイズ | 小(1×1): ポーション/鍵、中(1×2/2×1): 剣/杖、大(2×2): 盾/書物、特大(2×3): 大剣/弓 |
| 回転 | アイテムを90°回転して配置変更可能 |
| 自動整理 | ボタン一つでアイテムを自動最適配置 |
| コンテナ | バッグ内バッグ（入れ子インベントリ、1段階のみ） |

- 既存のInventory（MaxSlots/MaxWeight）をグリッドベースに移行
- IItem に Width/Height プロパティを追加
- GUI層はWPFのGridコントロールで描画
- 旧スロット方式との互換（設定で切替可能にする案も検討）

### 37.4 既存システムとの関連

- `Inventory` class: MaxSlots → GridWidth/GridHeight へ移行
- `IItem` interface: Width/Height プロパティ追加
- `IStackable`: スタックアイテムはサイズ不変
- `InventoryWindow.xaml.cs`（GUI層）: グリッド描画の実装
- `Player`: EquipmentSlots とグリッドインベントリの共存

### 37.5 対応バージョン

**Ver.prt.1.0**（グリッドシステム基盤・GUI実装）
