# 実装計画書 Ver.prt.0.15

## 概要
- **バージョン**: Ver.prt.0.15
- **テーマ**: バグ修正7件 & UIレイアウト全面オーバーホール
- **実装日**: 2025年
- **テスト結果**: Core 5,409 + GUI 148 = 5,557テスト全合格（GUIオートメーション11件除外）

## 実装タスク一覧

### タスク1: モンスタードロップアイテムのログ修正・実アイテム作成 ✅完了
**問題**: モンスターを倒した際に直接ドロップアイテムを入手したようなログが出ていた。ソウルジェム・素材採取はログのみで実際のアイテムが作成されていなかった。
**修正内容**:
- `TryDropItem()`のログメッセージを「を落とした！」→「が足元に落ちている」に変更（DropTableパス・フォールバックパスの両方）
- ソウルジェム: ログのみ → 実際の`KeyItem`作成 + `GroundItems.Add`で床にドロップ
- 素材採取（HarvestSystem）: ログのみ → 実際の`Material`アイテム作成 + `GroundItems.Add`で床にドロップ
**変更ファイル**: `GameController.cs` (TryDropItem L1383-1423, Soul gem L1525-1539, Harvest L1494-1517)

### タスク2: 隠し通路メッセージのダンジョン限定 ✅完了
**問題**: 「戦闘の衝撃で隠し通路が露わになった」メッセージがどこでも表示されていた。
**修正内容**:
- SecretRoomSystemの発見チェックにダンジョン限定ガード `!_worldMapSystem.IsOnSurface && !_isInLocationMap` を追加
**変更ファイル**: `GameController.cs` (L1519-1527)

### タスク3: ダンジョン入場のTキー統一 ✅完了
**問題**: シンボルマップでダンジョンのみ `Shift+>` キーで入場する仕様だったが、他の施設はTキーだった。
**修正内容**:
- `TryEnterTown()`にDungeonタイプの処理を追加（ダンジョン入場ロジックを移動）
- `TryDescendStairs()`からシンボルダンジョン入場処理を除去し、Tキー誘導メッセージに変更
**変更ファイル**: `GameController.cs` (TryEnterTown, TryDescendStairs)

### タスク4: グリッドインベントリの全アイテム表示修正 ✅完了
**問題**: グリッドインベントリに反映されないアイテムが存在していた。
**修正内容**:
- `PlaceItemsOnGrid()`の空きスペース未発見時の`break`を`continue`に修正。大きいアイテムが配置できない場合でも小さいアイテムの配置を継続するようになった。
**変更ファイル**: `InventoryWindow.xaml.cs` (PlaceItemsOnGrid L152-155)

### タスク5: インベントリ位置の永続化 ✅完了
**問題**: インベントリでソート・手動位置変更後にウィンドウを閉じると状態がリセットされていた。
**修正内容**:
- `MainWindow.xaml.cs`に`_inventoryGridPositions`辞書フィールドを追加
- `InventoryWindow`コンストラクタに`savedPositions`パラメータを追加
- `InventoryWindow.GetSavedPositions()`メソッドを追加
- `ShowInventoryDialog()`でウィンドウ間の位置受け渡しを実装
**変更ファイル**: `MainWindow.xaml.cs`, `InventoryWindow.xaml.cs`

### タスク6: スキルツリーに習得スキル一覧タブ追加 ✅完了
**問題**: スキルツリーに習得済みスキル一覧がなく、習得したスキルをスロットに装備できなかった。
**修正内容**:
- `SkillTreeWindow.xaml`に6番目の「習得済み」タブを追加
- 習得済みスキルをWrapPanel形式で一覧表示
- スキル選択で詳細表示、「スロットに装備」/「スロットから外す」ボタン
- スロット装備状態の視覚的フィードバック（緑枠・チェックマーク）
**変更ファイル**: `SkillTreeWindow.xaml`, `SkillTreeWindow.xaml.cs`

### タスク7: UIレイアウト全面オーバーホール ✅完了
**問題**: 全ステータス情報が1行のステータスバーに詰め込まれ、ゴールドなどの表示が見切れていた。
**修正内容**:
- 4隅レイアウトに全面書き換え:
  - **左上**: 場所・時刻情報（領地、階層、日時、時間帯、季節、天候、残り日数）
  - **左下**: ステータス（Lv, HP, MP, SP, 満腹, 渇き, 正気, 疲労, 衛生, 病気）
  - **右上**: 所持情報（ゴールド, EXP, 重量, 善悪, 仲間, 構え）
  - **右下**: メッセージログ
  - **マップ下**: PoE風スキルスロットアイコン（スキル名頭文字、スロット番号、ツールチップ）
- `UpdateStatusBar()`のスキルスロット表示をテキスト形式からアイコンBorder形式に変更
**変更ファイル**: `MainWindow.xaml`, `MainWindow.xaml.cs`
