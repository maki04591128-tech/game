## 1. 概要

Ver.prt.0.6 は Ver.prt.0.5 までに確立した「ゲーム完走フロー」を土台に、
10種の新規ゲームプレイシステムを追加し、死に戻りメカニクスの認識修正を行うフェーズである。

### 1.1 死に戻りメカニクス修正（重要）

従来の設計では死に戻り時にNPC記憶・噂・関係値が引き継がれる仕様であったが、
**正しい仕様**は以下の通り：

- 死に戻りは**同一キャラクターが開始地点に時間が巻き戻る形**で処理する
- 自身の痕跡や死体、NPCとの関係などは**全て残らない**（世界が完全リセット）
- キャラクター自身の内面的知識（スキル・魔法言語・宗教）は引き継がれる
- 正気度は死因に応じて減少する（従来通り）

> **変更点**: NpcMemorySystem の記憶・噂は死に戻り時にリセットされる。
> TransferData から NPC 関連データを除外する。

## 2. 方針

- 各システムは独立した静的クラスまたはインスタンスクラスとして `Core/Systems/` に配置
- 新規列挙型は `Core/Enums/Enums.cs` の末尾に追加
- 各システムに対応するテストファイルを `Core.Tests/` に作成
- 死に戻りメカニクス修正は既存コード（NpcMemorySystem・TransferData・設計書）に影響
- 既存テストが全て合格することを常に確認

**目的**: 死に戻りの仕様を「同一キャラクターの時間巻き戻し」に統一

**変更内容**:
1. NpcMemorySystem にリセットメソッド `Reset()` を追加
2. 死に戻り実行時に NpcMemorySystem.Reset() を呼び出す設計
3. 企画書・死に戻り設計書の記述を修正（NPC関係は引き継がれない）
4. TransferData の設計意図を明確化（キャラクター内面知識のみ）
