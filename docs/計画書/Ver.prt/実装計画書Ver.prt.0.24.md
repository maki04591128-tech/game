## 修正方針（ユーザー判断待ち）

ユーザーが各項目の「修正判断」欄に ✅（修正する）/ ❌（修正しない）/ 🔄（保留）を記入後、
確定した修正対象をまとめて実装する。

### 修正優先度の目安
1. **致命的**（6件）: 使用すると機能しないアイテム → 最優先で修正推奨
2. **高**（14件）: 設計と実装の明確な乖離 → 修正推奨
3. **中**（19件）: 違和感・バランス問題 → 選択的に修正
4. **低**（4件）: 軽微なテーマ不一致 → 余裕があれば修正
5. **設計課題**（1件）: アーキテクチャ改善 → 長期検討
