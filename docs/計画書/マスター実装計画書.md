## 2. プロジェクト構成（Phase A 時点・現状維持）

```
RougelikeGame.sln
├── src/
│   ├── RougelikeGame.Core/        # ドメイン層（エンティティ、値オブジェクト、アイテム、マップ、AI）
│   ├── RougelikeGame.Engine/      # エンジン層（戦闘、ダメージ計算、属性、状態異常、ターン管理）
│   ├── RougelikeGame.Data/        # データ層（魔法言語データベース、呪文解析）
│   ├── RougelikeGame.Gui/         # WPF GUI（GameController、描画、ウィンドウ群）
│   └── RougelikeGame.App/         # コンソールアプリ（InteractiveGameDemo）
└── tests/
    ├── RougelikeGame.Core.Tests/  # Core ユニットテスト
    └── RougelikeGame.Gui.Tests/   # Gui ユニットテスト
```

**方針（2026-04-17 改訂）**：コードは削除せず、旧仕様を**保全しつつ**新仕様を追加実装する非破壊アプローチを採用します。Phase B 以降で「休眠」システムの `GameController` 等からの参照を解除し、新企画書 §10 に従って表面上は約 28 システム構成に見える状態を作ります。詳細は `docs/流用資産マップ.md` を参照。
