## 5. 更新ルール

1. **新しいドキュメントを追加した場合**: 本ファイルのセクション1の一覧とセクション2の関連図を更新する
2. **既存ドキュメントの構成を大きく変更した場合**: セクション3の関連性テーブルを更新する
3. **「ドキュメントブラッシュアップ」実施時**: 全ドキュメントの情報不足・誤りを確認し、追記・修正・新規作成を行う

## 6. 用語集

| 用語 | 説明 |
|------|------|
| BSP法 | Binary Space Partitioning。ダンジョン自動生成アルゴリズム |
| FOV | Field of View。プレイヤーの視野範囲計算（レイキャスティング方式） |
| ターン | ゲーム内の最小時間単位。60ターン=1分（1ターン=1秒） |
| 正気度 | 死に戻り時に減少するメンタルパラメータ（0〜100）。6段階で行動に影響 |
| 満腹度 | 時間経過で減少する空腹パラメータ（0〜100）。0で餓死 |
| ルーン語 | 魔法言語の語彙体系。組み合わせで魔法効果が決定 |
| 素性 | キャラクター作成時に選ぶ背景設定。クリア条件が変化 |
| 死に戻り | HP0等で発生するリスポーン。知識は引き継がれるが正気度が減少 |
| GameTime | ゲーム内暦。冒険歴1024年開始、12ヶ月×30日制 |
| TurnCosts | 行動別ターン消費コスト定数クラス。移動=1、攻撃=3等 |
| GameConstants | ゲームバランス定数クラス。MaxLevel=50、MapWidth=80等 |
| DungeonMap | ダンジョンマップクラス。タイル管理・FOV計算を担当 |
| GameController | GUI層のゲームロジック制御クラス。入力→処理→描画の中核 |
| GameRenderer | Canvas描画クラス。マップ・エンティティ・ミニマップの描画 |
| CombatSystem | 戦闘処理クラス。ダメージ計算・命中/回避/会心判定 |
| StatusEffect | 状態異常クラス。毒/麻痺/睡眠等のティック処理 |
| CharacterCreation | キャラクター作成システム。RaceDefinition/ClassDefinition/BackgroundDefinition |
| TransferData | 死に戻り時の知識引き継ぎデータ。スキル・魔法言語・宗教・信仰度等 |
| RaceDefinition | 種族定義（10種）。ステータス補正・HP/MP補正・特性・経験値倍率 |
| ClassDefinition | 職業定義（10種）。ステータス補正・HP/MP補正・初期スキル |
| BackgroundDefinition | 素性定義（10種）。初期ゴールド・ステータス補正 |
| ExecuteRebirth | 死に戻り処理。プレイヤー再作成・知識引き継ぎ・初期装備付与 |
| WPF | Windows Presentation Foundation。GUIフレームワーク |
| Canvas | WPFの描画コンテナ。タイルベースのマップ描画に使用 |
| BalanceConfig | ゲームバランス設定クラス。階層/ランク別倍率、ドロップ率、ショップ倍率等 |
| DropTableSystem | ドロップテーブルシステム。14種のデフォルトテーブル、ルート生成・ステータススケーリング |
| CraftingSystem | 合成・強化システム。10レシピ、強化+10、エンチャント |
| TutorialSystem | チュートリアルシステム。18ステップ/18トリガー、自動完了機能 |
| WorldMapSystem | 世界マップシステム。6領地、領地間移動、レベル制限 |
| EnemyRank | 敵ランク列挙型。Common/Elite/Rare/Boss/HiddenBossの5段階 |
| TitleWindow | タイトル画面。ニューゲーム/コンティニュー/設定/終了の選択、キーボードナビゲーション |
| GameSettings | ゲーム設定モデル。音量3種/フォントサイズ/ウィンドウサイズ、JSON永続化 |
| IAudioManager | オーディオ管理インターフェース。BGM/SE再生・停止・音量制御 |
| AudioManager | WPF MediaPlayer実装。BGMループ、SE同時8再生、重複抑制 |
| ObjectPool | 汎用オブジェクトプール。ConcurrentBagベース、スレッドセーフ |
